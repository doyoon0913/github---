# 📌 C 언어 포인터(Pointer) 핵심 요약

C 언어에서 가장 중요한 **포인터(Pointer)**의 기본 개념과 주의할 점을 정리한 문서입니다.

---

## 1. 포인터의 기본 개념
포인터는 메모리의 **주소값(Address)**을 저장하는 특별한 변수입니다.

* **`&` 연산자 (주소 연산자):** 변수가 메모리 상에서 몇 번지에 위치해 있는지 주소를 찾아냅니다.
* **`*` 연산자 (역참조 연산자):** 포인터가 가리키고 있는 주소로 직접 찾아가서 그곳에 저장된 실제 데이터를 읽거나 변경합니다.

---

## 2. 포인터 출력 시 주의할 점 (`%p` vs `%d`)
포인터 변수에 저장된 주소값을 출력할 때는 컴파일러 경고 없이 안전하게 출력하기 위해 올바른 서식 지정자를 사용해야 합니다.

* **`%p` (올바른 방법):** 포인터가 가리키는 메모리 주소를 16진수 형태(예: `0x7ffee3b4b7bc`)로 안전하게 출력합니다.
* **`%d` (잘못된 방법):** 주소값을 강제로 10진수 정수로 표현하려고 하면, 시스템 환경(32비트/64비트)에 따라 데이터가 잘리거나 컴파일러 경고가 발생합니다.
* **`*포인터`와 `%d` 조합:** 주소값이 아니라 **그 주소에 들어있는 실제 정수 값**을 출력하고 싶을 때만 `%d`와 역참조 연산자(`*`)를 함께 사용합니다.

---

## 3. 포인터 증감 연산의 규칙
포인터 변수에 `++`나 `--` 같은 증감 연산을 수행할 때, 주소값은 단순히 1씩 증가하거나 감소하지 않습니다. **포인터가 가리키는 자료형(Data Type)의 크기만큼** 건너뜁니다.

예를 들어, 임의의 주소 `10000`번지가 있다고 가정할 때의 연산 결과는 다음과 같습니다.

| 포인터 타입 | 가리키는 자료형의 크기 | 연산 (`++`) | 연산 후 주소값 변경 |
| :--- | :---: | :---: | :--- |
| **`char *`** | 1 Byte | `pc++;` | **10001** (10000 + 1) |
| **`int *`** | 4 Byte | `pi++;` | **10004** (10000 + 4) |
| **`double *`** | 8 Byte | `pd++;` | **10008** (10000 + 8) |

> 💡 **핵심:** 컴파일러가 메모리를 효율적으로 가리키기 위해, 다음 데이터가 시작되는 위치로 주소를 자동 이동시키는 것입니다.

---

## ⚠️ 주의: 잘못된 주소 접근 (Segmentation Fault)
프로그램 실행 중 운영체제(OS)가 안전하게 할당해 주지 않은 임의의 주소(예: `10000`번지 등)를 포인터에 강제로 대입하는 것은 주소 연산 원리를 이해하기 위한 테스트용으로만 사용해야 합니다.

만약 이 임의의 주소에 대고 **역참조 연산(`*`)을 통해 값을 읽거나 쓰려고 시도**하면, 운영체제의 메모리 보호 시스템에 의해 프로그램이 즉시 강제 종료되는 **Segmentation Fault(세그멘테이션 오류)**가 발생합니다.
